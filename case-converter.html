Everything You Need to Know About Case Conversion

Understanding case styles helps you pick the right format for every context—code, content, URLs, and beyond.

🔠

UPPER CASE & lower case

The most fundamental conversions. UPPER CASE (all capitals) is used for constants in programming, legal notices, and design emphasis. lower case is standard for URLs, email addresses, and many modern design styles that favour minimal capitalisation.

📰

Title Case

Capitalises the first letter of every significant word, following standard English title conventions. Short prepositions (in, of, to), articles (a, an, the), and coordinating conjunctions remain lowercase unless they open the title. Used for headlines, article titles, and headings.

💬

Sentence case

Capitalises only the first letter of the first word and proper nouns—exactly as you'd write a normal sentence. This is the default for body copy, email subject lines, button labels in modern UI design, and general writing where heavy capitalisation would feel formal or dated.

⚙️

camelCase & PascalCase

Both remove spaces and capitalise word boundaries. camelCase starts with a lowercase letter (myVariable) and is standard in JavaScript, Java, and Swift. PascalCase starts every word with a capital (MyClass) and is used for class names, component names in React, and C# conventions.

🐍

snake_case

Replaces spaces with underscores and lowercases all letters (my_variable_name). The dominant convention in Python for variables, functions, and module names. Also common in database column names and REST API field names where hyphens aren't allowed.

🔗

kebab-case

Replaces spaces with hyphens and lowercases all letters (my-variable-name). The standard for HTML attribute values, CSS class names, file names, and URL slugs. Preferred in Lisp and some CSS-in-JS naming conventions. Sometimes called "slug case" in the context of URLs.
